# Mint-AI REST API 设计说明

## 1. 概述

本设计面向「输入视频、一键成片」的后端服务，采用 **FastAPI** 提供 RESTful API，运行时自动生成 **OpenAPI 文档**（Swagger UI / ReDoc），便于联调与契约对齐。

| 目标 | 说明 |
|------|------|
| 文档 | 服务启动后可通过浏览器访问交互式 API 文档 |
| 版本 | URL 路径级版本号，避免破坏性变更影响旧客户端 |
| 参数 | 使用 Pydantic 模型做 **严格类型与约束校验**（请求体、查询参数） |
| 并发 | 路由处理函数采用 **async**，与异步 I/O（HTTP 拉流、任务队列等）对接 |

**包管理**：工程统一使用 **[uv](https://github.com/astral-sh/uv)** 管理 Python 版本、虚拟环境与依赖（`pyproject.toml` + 锁文件）。

---

## 2. API 版本与基础路径

- **策略**：在路径中包含主版本号，例如 `v1`。主版本升级表示不兼容的 URL、字段或语义变更。
- **基础前缀**：`/api/v1`

后续若发布 `v2`，可并行暴露 `/api/v2/...`，`v1` 在约定期内继续维护。

---

## 3. 统一响应结构

所有业务接口在 HTTP 200 且业务成功时，响应体采用统一外层结构（便于前端与监控解析）：

| 字段 | 类型 | 说明 |
|------|------|------|
| `code` | `integer` | 业务码，`0` 表示成功；非 `0` 表示失败（与 HTTP 状态码配合使用时可再约定） |
| `message` | `string` | 人类可读说明 |
| `data` | `object \| null` | 业务数据；无数据时为 `null` 或空对象（见各接口） |

> **说明**：具体「失败时 HTTP 状态码取 4xx/5xx 还是仍返回 200 + code」可在实现阶段固定一条团队约定；本设计文档以接口字段与语义为主。

---

## 4. 接口设计

### 4.1 提交成片任务

| 项目 | 说明 |
|------|------|
| **方法** | `POST` |
| **URL** | `/api/v1/tasks` |
| **Content-Type** | `application/json` |

**请求体（JSON）**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `video_url` | `string` | 是 | 待处理视频的访问地址（建议约束为 `http`/`https`，长度上限在实现时用 Pydantic 校验） |

**请求示例**

```json
{
  "video_url": "https://example.com/video.mp4"
}
```

**成功响应示例**（HTTP 200）

```json
{
  "code": 0,
  "message": "success",
  "data": {
    "task_id": "xxx"
  }
}
```

**`data` 字段**

| 字段 | 类型 | 说明 |
|------|------|------|
| `task_id` | `string` | 任务唯一标识，用于后续查询 |

---

### 4.2 查询任务状态 / 结果

| 项目 | 说明 |
|------|------|
| **方法** | `GET` |
| **URL** | `/api/v1/tasks` |
| **Query** | `task_id`（必填） |

**请求示例**

```http
GET /api/v1/tasks?task_id=xxx
```

**Query 参数**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `task_id` | `string` | 是 | 提交任务时返回的 `task_id` |

**成功响应示例**（HTTP 200）

```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```

> **说明**：当前阶段 `data` 可为空对象 `{}`；后续可在不改变外层结构的前提下扩展，例如增加 `status`、`progress`、`result_url` 等字段，并在本文档中迭代版本说明。

---

### 4.3 文档与健康检查（建议）

| 项目 | 说明 |
|------|------|
| OpenAPI JSON | FastAPI 默认 `/openapi.json` |
| Swagger UI | 默认 `/docs` |
| ReDoc | 默认 `/redoc` |
| 健康检查（可选） | `GET /health`，用于负载均衡探活，返回简单 JSON 即可 |

以上路径若与网关前缀拼接，需在部署文档中说明对外暴露的完整 URL。

---

## 5. 严格参数校验（实现约定）

实现阶段建议遵守以下约定，以满足「严格参数检查」：

1. **请求体**：使用 Pydantic `BaseModel`，对 `video_url` 做类型、`min_length`、URL scheme（如仅允许 `http`/`https`）等约束。
2. **查询参数**：`task_id` 使用 Pydantic 模型或 FastAPI `Query(...)`，禁止空字符串（若业务不允许）。
3. **校验失败**：返回 **422**，响应体为 FastAPI 默认验证错误结构；若产品要求与「统一 `code`/`message`」完全一致，可在后续迭代中增加全局异常处理器（设计层面预留即可）。

---

## 6. 异步接口（实现约定）

- 业务路由使用 `async def`，内部对网络请求、任务投递等尽量使用异步库或 `asyncio.to_thread` 包装阻塞调用，避免阻塞事件循环。
- CPU 密集或重 FFmpeg 流水线可放到 **任务队列 / 独立 worker**，API 层只负责受理与查询（与本设计中的「先不实现具体成片逻辑」一致）。

---

## 7. 工程目录结构（建议）

以下为基于 FastAPI + uv 的推荐布局，实现时可按模块微调，但保持「版本路由、模型、服务分层」清晰即可。

```text
mint-ai/
├── pyproject.toml          # 项目元数据与依赖（uv 使用此文件）
├── uv.lock                 # 依赖锁定（由 uv 生成）
├── README.md
├── docs/
│   └── 设计文档.md
├── src/
│   └── mint_ai/            # 包名示例，实现时可与 pyproject 中 package 名一致
│       ├── __init__.py
│       ├── main.py         # FastAPI app 创建、挂载路由、CORS、生命周期
│       ├── api/
│       │   ├── __init__.py
│       │   └── v1/
│       │       ├── __init__.py
│       │       ├── router.py     # 聚合 v1 路由
│       │       └── tasks.py      # tasks 相关端点（POST/GET）
│       ├── schemas/
│       │   ├── __init__.py
│       │   ├── common.py       # 统一响应体、业务码枚举等
│       │   └── tasks.py        # Task 创建请求、Task 查询响应等 Pydantic 模型
│       ├── services/
│       │   ├── __init__.py
│       │   └── task_service.py # 任务创建、查询（后续接存储/队列）
│       └── core/
│           ├── __init__.py
│           └── config.py       # 环境变量与配置（如 BASE_URL、超时）
├── tests/
│   ├── __init__.py
│   └── api/
│       └── test_tasks.py
└── .python-version           # 可选，指定 Python 版本供 uv 使用
```

**分层说明（简要）**

| 目录/文件 | 职责 |
|-----------|------|
| `api/v1/` | 仅 HTTP 层：路径、依赖注入、调用 service，保持薄 |
| `schemas/` | 请求/响应 Pydantic 模型，供 OpenAPI 生成与严格校验 |
| `services/` | 业务逻辑与与存储/队列交互（当前可为占位实现） |
| `core/` | 配置、常量、共享异常类型等 |

---

## 8. uv 使用说明（工程约定）

- **初始化项目**：在仓库根目录使用 `uv init` 或手写 `pyproject.toml`，依赖通过 `uv add fastapi uvicorn[standard] pydantic-settings ...` 添加。
- **运行**：`uv run uvicorn mint_ai.main:app --reload`（具体模块路径以 `pyproject` 中包名为准）。
- **锁文件**：将 `uv.lock` 纳入版本控制，保证 CI 与生产构建可复现。
